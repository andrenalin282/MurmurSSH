# Open Source Guidelines – MurmurSSH

## Project Nature
MurmurSSH is intended to be open source and community-friendly.

## Code Expectations
- keep code readable
- prefer explicit naming
- avoid clever but obscure patterns
- write comments only where they add real value
- avoid unnecessary indirection

## Community Expectations
- structure the project so new contributors can navigate it easily
- prefer conventional tooling
- avoid hidden project-specific magic

## Documentation Expectations
When creating project files, include or prepare:
- README.md
- basic setup instructions
- contribution guidance when appropriate

## Scope Discipline
Do not expand the project into a broad SSH platform.
This project is intentionally small and focused.

## Review Mindset
Every addition should be judged by:
- does it serve the MVP?
- does it keep the project simple?
- does it help maintainability?
- is it worth the added complexity?
