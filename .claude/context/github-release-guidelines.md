# Repository Working Rules

## Commit convention
- feat: new functionality
- fix: bugfix
- docs: documentation only
- refactor: internal improvement without behavior change
- chore: maintenance
- build: packaging/release changes
- ci: workflow changes
- test: tests

## Release versioning
- Use SemVer
- Tags use format vX.Y.Z
- Initial public releases stay in 0.x

## Initial release artifacts
- .deb (amd64)
- .AppImage (amd64)

## Collaboration style
- Prefer small focused changes
- Respect existing structure
- Avoid unnecessary rewrites
- Keep documentation concise and practical
